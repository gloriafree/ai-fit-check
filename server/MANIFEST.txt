================================================================================
API ENDPOINTS
================================================================================

GET /                        - Root with API info
GET /health                  - Server health and status

POST /api/person            - Upload/update person full-body photo
GET /api/person             - Retrieve current person photo

POST /api/tryon             - Perform virtual try-on with clothing image
                            - Returns base64 try-on result

GET /api/wardrobe           - List all saved wardrobe items
POST /api/wardrobe          - Save try-on result to wardrobe
DELETE /api/wardrobe/{id}   - Delete wardrobe item

================================================================================
PIPELINE FLOW
================================================================================

Try-On Request:
  1. User uploads clothing image
  2. Server loads saved person image
  3. Segment clothing (remove background)
  4. Call Fashn.ai API for virtual try-on
  5. Poll for result (up to 120s)
  6. Return base64-encoded result
  Total time: 15-40 seconds

Wardrobe Save:
  1. Generate UUID for item
  2. Save PNG image to disk
  3. Create metadata JSON
  4. Return item info to client

================================================================================
PERFORMANCE NOTES
================================================================================

Try-On Timing:
  - Segmentation: 1-5s (GPU) or <100ms (CPU with rembg)
  - Fashn.ai API: 10-30s (depends on queue)
  - Total: 15-40s per request

Memory Usage:
  - Idle: ~200MB
  - Per request: 100-500MB
  - Peak: 2-4GB (with Grounded SAM model)

Recommended Specs:
  - RAM: 8GB minimum
  - GPU: NVIDIA with CUDA support (optional)
  - Storage: 50GB+ for wardrobe items

Scaling:
  - Single worker: ~1.5 try-ons/minute
  - Multiple workers: Limited by Fashn.ai API
  - Use load balancer for distribution
